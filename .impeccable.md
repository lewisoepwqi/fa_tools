# Design Context — FA Tools 财务自动化工具包

> 本文件为 frontend-design 等 design skills 的设计上下文，由 teach-impeccable 流程确立。
> 修改本文件即改变全站设计基准。

## 目标用户与使用场景

两类角色共用一套系统，需按场景照顾不同诉求：

- **财务专员**：日常高频处理。上传银行流水 → 预览转换结果 → 修正异常项 → 导出日记账。
  熟悉会计术语，追求**快、准、少点击**。需要信息密集的工作台、快捷操作、批量处理。
- **财务主管**：周期性配置与审核。管理银行/日记账模板、映射方案、规则；审核批次、
  确认结果、导出记录。关注**全局与合规**，需要可读性、版本/审计追溯、批量审核。

设计上：专员工作流页（上传/批次详情）信息密集、操作前置；配置与审核页（模板/规则/审计）
强调可读性与追溯。

## 品牌调性：中大咨询集团企业 VI · 专业严谨

采用集团统一视觉识别（源自 2026 集团 PPT 模板 token，见 `tokens.css`）。

### 配色
- **品牌红 `#b5141d`**（主色 / accent / 强调条）——主操作、关键强调、左侧装饰条
- **品牌蓝 `#133f8e`**（结构色 / 深海军蓝）——结构性元素、次级强调、链接
- **暖中性**：文字 `#1D1B1B`（带棕黑偏移，非纯黑）/ 次级文字 `#6f6864` /
  边框 `#d8d0ca` / 背景 `#f5f2ee`（暖纸色，非纯白）
- 禁止纯黑纯白（一律走暖中性 token）；禁止 cyan-on-dark / 紫蓝渐变 / 霓虹。

### 字体
- **中文标题/品牌字标**：楷体（KaiTi，`@font-face` 加载本地 simkai.ttf）——用于页面主标题、品牌字标，体现集团气质
- **中文正文/UI**：微软雅黑 / PingFang SC（保证可读性，楷体仅作标题装饰）
- **西文/数字**：Lato（Calibri 的 web 替代，Google Fonts）——金额/编号用 Lato + tabular-nums
- 禁止 Inter/Roboto 作主字体。

### 形态与节奏
- **圆角遵循 frontend-design skill**：克制偏小、精致——卡片 6px、按钮/输入 4px、标签 2px，体现金融精密感（不参考 tokens.css 的 0 圆角）
- **左侧红色强调条**（约 3×20px 量级）是核心视觉记忆点，用于区块标题前点睛
- 暖色分隔线（`#d8d0ca`），非冷灰
- 讲究的间距节奏（token space-1..9），左对齐为主
- 现代前端手法：克制的卡片阴影、流畅过渡、tabular-nums 数字对齐、合理留白

## 约束

- 技术栈：React 18 + TypeScript + Ant Design v5 + Vite。
  在 Ant Design 之上叠加自定义主题 token 与样式，而非脱离它另造组件库。
- 中文为主界面语言。
- 必须保留现有路由结构与 API 契约（重构视觉与交互，不改后端）。

## 差异化记忆点

让人记住的"一件事"：**精密如金融终端的数据密度，却有手工装订般的版式秩序**——
表格、数字、状态用克制的金色与墨绿点睛，让财务一眼读出关键金额与异常。
